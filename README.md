# ZabServer - бекэнд для [ZabKLI](https://github.com/MrHantur/ZabKLI)

## Реализовано:
• Аутентификация: регистрация, вход, refresh-токены (JWT)
• Ролевая модель: viewer - contributor - editor - admin
(contributor ещё сыроват, для него есть роль в коде, а также раздел предложений изменений, но он пока не используется и может претерпеть изменения)
• Раздел олимпиад: ильтрация по дате, предмету, классам, уровню
(на деле используется получение всех олимпиад, фильтрация пока не имеет смысла, может использоваться в будущем)
• Расписание: изменение данных о уроках, фильтрация по дню недели и классу
• Система предложений: реализована, однако не используется. Нужна на будущее, чтобы находить людей для повышения до роли editor
• Анкетирование: сбор данных о удовлетворённости приложением
• Rate Limiting: ограничено количество запросов от пользователя, чтобы не уронили сервер

## Ознакомиться с API можно по [ссылке](http://zab.mrhantur.su:1717/docs)
Страница Zab API создана автоматически с помощью FastAPI

## Безопасность
• Пароли хешируются через bcrypt
• JWT с раздельными access/refresh токенами
• Валидация входных данных через Pydantic
• Защита от инъекций (SQLAlchemy ORM + параметризованные запросы)
