# ZabServer - бекэнд для [ZabKLI](https://github.com/MrHantur/ZabKLI)

## Реализовано
1. Аутентификация: регистрация, вход, refresh-токены (JWT)
2. Ролевая модель: viewer - contributor - editor - admin (contributor ещё сыроват, для него есть роль в коде, а также раздел предложений изменений, но он пока не используется и может претерпеть изменения)
3. Раздел олимпиад: ильтрация по дате, предмету, классам, уровню (на деле используется получение всех олимпиад, фильтрация пока не имеет смысла, может использоваться в будущем)
4. Расписание: изменение данных о уроках, фильтрация по дню недели и классу
5. Система предложений: реализована, однако не используется. Нужна на будущее, чтобы находить людей для повышения до роли editor
6. Анкетирование: сбор данных о удовлетворённости приложением
7. Rate Limiting: ограничено количество запросов от пользователя, чтобы не уронили сервер

## Ознакомиться с API можно по [ссылке](http://zab.mrhantur.su:1717/docs)
Страница Zab API создана автоматически с помощью FastAPI

## Безопасность
1. Пароли хешируются через bcrypt
2. JWT с раздельными access/refresh токенами
3. Валидация входных данных через Pydantic
4. Защита от инъекций (SQLAlchemy ORM + параметризованные запросы)
